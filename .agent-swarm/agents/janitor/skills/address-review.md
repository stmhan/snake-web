# 코드 리뷰 피드백 처리

## 워크플로우

### 1. 피드백 수집

리뷰 요약, 인라인 코멘트, 일반 코멘트를 모두 수집한다.

### 2. 분류

| 분류 | 기준 | 처리 |
|------|------|------|
| MUST FIX | 리뷰어가 명시적으로 변경을 요청한 항목 | 반드시 수정 |
| SHOULD FIX | 리뷰어의 제안 중 코드 품질을 높이는 항목 | 수정 권장 |
| SKIP | 이미 해결되었거나 자동화 범위 밖의 질문 | 건너뜀 |

### 3. 수정 적용

MUST FIX → SHOULD FIX 순서로 처리한다.
각 수정은 리뷰어가 요청한 범위 내에서만 수행한다.

### 4. 검증

수정 후 프로젝트의 테스트 명령어를 실행하여 기존 테스트가 통과하는지 확인한다.
리뷰어가 테스트 추가를 요청한 경우 테스트를 작성한다.

### 5. 커밋

모든 수정을 하나의 커밋으로 묶는다. 커밋 메시지는 한국어로 작성한다:
```
fix: 코드 리뷰 피드백 반영 - {변경 요약}
```

## 수정 범위 규칙

**수정하는 것:**
- 리뷰어가 지적한 코드만 수정
- 리뷰어가 요청한 테스트만 추가
- 리뷰어가 지적한 네이밍, 구조 변경만 적용

**수정하지 않는 것:**
- 리뷰와 관련 없는 코드 리팩토링
- 리뷰에서 언급되지 않은 새 기능 추가
- 리뷰 범위 밖의 코드 스타일 변경
- 리뷰어가 승인한 코드의 재수정

이 규칙은 수정 범위를 제한하여 리뷰-수정 순환이 수렴하도록 한다.
