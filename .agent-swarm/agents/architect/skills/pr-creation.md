# PR 생성

## 필수 규칙

- PR은 반드시 **현재 작업 중인 리포지토리**에 생성한다
- PR 본문은 반드시 **한국어**로 작성한다
- `gh pr create` 명령어를 사용하여 PR을 생성한다

## PR 제목 형식

```
feat: {이슈 번호} {기능 요약}
```

예시: `feat: #42 사용자 인증 기능 추가`

## PR 본문 구조

```markdown
## 요약
변경 사항에 대한 간략한 설명 (1-3줄)

## 관련 이슈
- #{이슈 번호}

## 변경 내용
- 변경된 파일과 내용을 구체적으로 기술
- 새로 추가된 기능 설명
- 수정된 버그 설명

## 테스트
- 실행한 테스트 명령어
- 테스트 결과 요약
- 수동 테스트 항목 (해당 시)

## 스크린샷 (해당 시)
UI 변경이 있는 경우 스크린샷 첨부
```

## PR 생성 명령어

```bash
gh pr create \
  --title "feat: #{이슈번호} {요약}" \
  --body "$(cat <<'EOF'
## 요약
...

## 변경 내용
...

## 테스트
...
EOF
)" \
  --base main
```

## 주의사항

- `--base` 플래그로 올바른 대상 브랜치 지정
- 포크가 아닌 같은 리포지토리에서 PR 생성
- 레이블이 있으면 `--label` 플래그로 추가
- 드래프트 PR이 적절한 경우 `--draft` 플래그 사용
