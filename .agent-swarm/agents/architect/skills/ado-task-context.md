# ADO 작업 컨텍스트

Azure DevOps 작업 항목에서 기능을 구현할 때, 먼저 전체 컨텍스트를 수집한다.

## 계층 탐색

1. 주어진 작업 항목 ID에서 시작
2. 계층 구조를 **위로** 탐색: Task -> User Story -> Feature
3. 최상위에서 **아래로** 탐색하여 모든 하위 항목과 형제 Task 확인
4. 관련 항목에 연결된 GitHub PR 확인

## 추출할 정보

- **요구사항**: User Story의 인수 기준 (Acceptance Criteria)
- **설계 문서**: 첨부된 문서 또는 설명
- **구현 세부사항**: 상위 Feature의 기술 노트
- **관련 작업**: 형제 Task들과 그 상태
- **기존 PR**: 이 기능에 대해 이전에 제출된 코드

## 구현 계획

컨텍스트 수집 후:
1. 요구사항 요약
2. 변경이 필요한 파일 식별
3. 다른 Task에 대한 의존성 나열
4. 단계별 구현 계획 수립
5. 테스트를 먼저 작성 (TDD), 그 다음 구현
